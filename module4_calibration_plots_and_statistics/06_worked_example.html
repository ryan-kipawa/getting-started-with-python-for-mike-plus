Worked Example: End-to-End Validation

This section walks through a complete validation workflow using ModelSkill, from preparing data to interpreting results.

Scenario Overview

We’ll validate a hypothetical river model’s water level and discharge at two gauge locations. Observation data are assumed to be available (e.g., in dfs0 or csv files), and model results are from a MIKE+ res1d file. For this example, we’ll use Pandas DataFrames to represent this data.

First, let’s import the necessary libraries.

import pandas as pd
import modelskill as ms

Step 1: Preparing Observation Data

Observation data, typically from sources like dfs0 files or CSVs, needs to be loaded and structured as Observation objects. Key parameters include name (for identification) and unit.

# Create dummy observation data for two locations
obs_wl_data = pd.DataFrame(
    {"WaterLevel": [1.0, 1.2, 1.1, 1.3, 1.0]},
    index=pd.date_range("2023-01-01", periods=5, freq="h")
)
obs_q_data = pd.DataFrame(
    {"Discharge": [10.0, 12.5, 11.3, 13.2, 9.8]},
    index=pd.date_range("2023-01-01", periods=5, freq="h")
)

obs1 = ms.Observation(obs_wl_data, item="WaterLevel", unit="m", name="Gauge_Alpha_WL")
obs2 = ms.Observation(obs_q_data, item="Discharge", unit="m^3/s", name="Gauge_Beta_Q")

observations_list = [obs1, obs2]

Step 2: Preparing Model Result Data

Model results, often extracted from res1d files for MIKE+ models, are prepared as ModelResult objects. Ensure name, unit, and correct item strings are used.

# Create dummy model result data corresponding to the observations
mod_wl_data = pd.DataFrame(
    {"Simulated_WL": [1.05, 1.15, 1.12, 1.28, 0.95]},
    index=pd.date_range("2023-01-01", periods=5, freq="h")
)

Step 3: Matching Data

Use modelskill.match() to pair Observation and ModelResult objects. Matching typically occurs based on the name attribute if provided and unique, or by the order of items in the lists.

comparer = ms.match(observations_list, model_results_list)

Step 6: Discussion and Interpretation

Based on the visual plots and quantitative skill scores, you can now discuss the overall model performance. For our hypothetical example: The time series plots would show how well the model captures the dynamics at each location. Scatter plots indicate the agreement, with points ideally clustering around the 1:1 line. KDE plots reveal if the model reproduces the observed data distribution.

Skill scores provide objective measures. For instance, a low Bias and RMSE for “Gauge_Alpha_WL” would suggest good performance for water level at that location. If “Gauge_Beta_Q” shows a higher MAE or lower NSE, it might indicate areas for model improvement concerning discharge simulation at that point. This structured validation helps identify strengths and weaknesses, guiding further model development or calibration efforts.
