Homework

These exercises will guide you through applying ModelSkill to validate a MIKE+ model. You will download a sample model and associated data, prepare observations and model results, compare them, and quantify the model’s performance.

General Instructions: * Create a new Jupyter Notebook for your solutions. * Import necessary packages at the beginning of your notebook: python import modelskill as ms import mikeio import mikeio1d import pandas as pd import matplotlib.pyplot as plt # For any custom plotting or plot adjustments

Exercise 1: Project Setup and Model Familiarization

  1. Create a new project folder on your computer specifically for this homework.
  2. Download the following files into your newly created project folder:
    • MIKE+ Model Package: [UrbanTestModel.zip](data/UrbanTestModel.zip){download="UrbanTestModel.zip"} (This zip file contains the MIKE+ model project.)
    • Flow Meter Data (Location A): [Flow_Obs_A.dfs0](data/Flow_Obs_A.dfs0){download="Flow_Obs_A.dfs0"}
    • Flow Meter Data (Location B): [Flow_Obs_B.dfs0](data/Flow_Obs_B.dfs0){download="Flow_Obs_B.dfs0"}
    • Rainfall Data (Model Input): [Rainfall_Input.dfs0](data/Rainfall_Input.dfs0){download="Rainfall_Input.dfs0"}
    • Reference Model Results: [Reference_Results.res1d](data/Reference_Results.res1d){download="Reference_Results.res1d"} (This can be used if you encounter issues running the model, or for initial setup.)
  3. Unzip the UrbanTestModel.zip file in your project folder. This should create a folder structure for the MIKE+ model.
  4. Open the MIKE+ model (e.g., UrbanTestModel.mplus or similar, located within the unzipped folder).
  5. Familiarize yourself with the model: ::: {.callout-note title=“About the UrbanTestModel (Details to be provided by instructor)”} [Placeholder: Your instructor will provide details about this model. This might include:
    • Type of System: Is it an urban drainage collection system, a river network, or something else?
    • Key Components: What are some of the main pipes/reaches, inlets, junctions, and outfalls?
    • Simulation Period: What time frame does the simulation cover?
    • Purpose: What is the model intended to simulate or analyze?
    • Observation Points: You will need to identify the model output locations (e.g., reach names and chainages, or node IDs) that correspond to “Flow Meter A” and “Flow Meter B”. For this exercise, let’s assume:
      • Flow Meter A corresponds to ‘Discharge’ in reach MSM_Link_2 at chainage 25.0 (or the closest grid point).
      • Flow Meter B corresponds to ‘Discharge’ in reach MSM_Link_5 at chainage 30.0 (or the closest grid point). You may need to explore the model setup in MIKE+ to confirm these locations or find the exact grid point chainages.] :::
  6. If you are comfortable with MIKE+, try running the simulation. Ensure that a new result file (e.g., ending in .res1d) is produced. For the subsequent exercises, you can use either your own generated result file or the provided Reference_Results.res1d.

Exercise 2: Preparing Observation Objects

  1. In your Jupyter Notebook, create a PointObservation object for the data from “Flow Meter A” (Flow_Obs_A.dfs0).
    • Name the PointObservation object obs_A.
    • Use the data parameter to point to the Flow_Obs_A.dfs0 file.
    • Specify the correct item from the dfs0 file (you might need to inspect the dfs0 file first, e.g., using mikeio.read("data/Flow_Obs_A.dfs0").items).
    • Set the name parameter of the PointObservation to "Flow_Meter_A". This will be used for labeling.
  2. Create a PointObservation object for the data from “Flow Meter B” (Flow_Obs_B.dfs0).
    • Name the PointObservation object obs_B.
    • Use the data parameter to point to the Flow_Obs_B.dfs0 file.
    • Specify the correct item.
    • Set the name parameter of the PointObservation to "Flow_Meter_B".
  3. For both obs_A and obs_B:
    • Print the object to see its summary (e.g., print(obs_A)).
    • Use the .plot() method to visualize the time series data (e.g., obs_A.plot()).

Exercise 3: Preparing ModelResult Objects

  1. You will use a model result file (either the Reference_Results.res1d or the one you generated in Exercise 1).
  2. For the location corresponding to “Flow Meter A” (e.g., ‘Discharge’ in reach MSM_Link_2 at chainage 25.0):
    • Use mikeio1d.open() to open your chosen res1d file.
    • Extract the ‘Discharge’ time series for this specific location.
    • Save this extracted time series to a new dfs0 file named model_discharge_A.dfs0. (Alternatively, convert it directly to a Pandas DataFrame).
      • Hint: When mikeio1d saves to dfs0, the item name in the dfs0 file will typically be descriptive (e.g., reach:Discharge:MSM_Link_2:25.000).
    • Create a PointModelResult object.
      • Name this object mod_A.
      • Use the data parameter to point to model_discharge_A.dfs0 (or the DataFrame).
      • Specify the item name that was created in your model_discharge_A.dfs0 (or the column name if using a DataFrame).
      • Set the name parameter of the PointModelResult to "MIKE_Plus_Sim".
  3. Repeat a similar process for the location corresponding to “Flow Meter B” (e.g., ‘Discharge’ in reach MSM_Link_5 at chainage 30.0):
    • Extract the ‘Discharge’ time series from the res1d file.
    • Save it to a new dfs0 file named model_discharge_B.dfs0 (or convert to a DataFrame).
    • Create a PointModelResult object.
      • Name this object mod_B.
      • Use model_discharge_B.dfs0 (or the DataFrame) for the data parameter.
      • Specify the correct item.
      • Set the name parameter of this PointModelResult to "MIKE_Plus_Sim" (note: this is the same model name as mod_A because they come from the same simulation run).
  4. For both mod_A and mod_B:
    • Print the object summary.
    • Use the .plot() method to visualize the model result time series.

Exercise 4: Matching Data and Individual Comparisons

  1. Match the observation obs_A with the model result mod_A using ms.match(). Store the result in a Comparer object named comparer_A.
  2. Match obs_B with mod_B. Store the result in a Comparer object named comparer_B.
  3. For comparer_A:
    • Generate and display a time series plot: comparer_A.plot.timeseries().
    • Generate and display a scatter plot: comparer_A.plot.scatter().
    • Calculate and display the default skill table: comparer_A.skill().
  4. Repeat step 3 for comparer_B.
  5. Look at the skill tables. Which metrics indicate better or worse performance for each location? (This is a qualitative observation for now).

Exercise 5: Aggregated Analysis with ComparerCollection

  1. Create a ComparerCollection named cc by providing a list containing comparer_A and comparer_B.
  2. Display the cc object (e.g., print(cc)). Check the output of cc.obs_names and cc.mod_names.
  3. Generate and display an aggregated scatter plot for the entire collection: cc.plot.scatter(). How does this compare to the individual scatter plots?
  4. Generate and display an aggregated histogram of residuals (Model - Observation) for the collection: cc.plot.residual_hist().
  5. Calculate and display the skill table for the collection using cc.skill(). This table should show skill scores for each individual observation point (Flow_Meter_A and Flow_Meter_B).
    • Modify the call to cc.skill() to show only the ‘rmse’, ‘bias’, ‘nse’, and ‘kge’ metrics.
  6. Calculate and display the mean skill table for the collection using cc.mean_skill(). This table shows the average skill across all observation points.
    • Modify the call to cc.mean_skill() to show only ‘rmse’, ‘bias’, ‘nse’, and ‘kge’.
  7. Using the cc.score() method, retrieve and print the mean ‘rmse’ and mean ‘kge’ for the "MIKE_Plus_Sim" model across both observation points.

Good luck! These exercises will solidify your understanding of how to use ModelSkill for practical model validation tasks.
